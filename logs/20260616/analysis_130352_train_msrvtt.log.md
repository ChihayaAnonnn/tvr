## 关键发现

1. **detach 成功隔离了梯度冲突** — ret_gap 从 16.7 恢复到 26.3（接近 Dir1_colwise 的 27.5），表征学习未被污染
2. **MLP 坍缩为常量** — cnf_d = cnf_off = 0.59，gap = 0.000 贯穿全程，未学习到任何对角/非对角区分
3. **T2V 仍低于 baseline** — 48.6，−0.8 vs 49.4。confidence 退化为矩阵整体乘常数（softmax 中约掉），效果等同于无加权
4. **根因：输入信号无区分度** — `anchor_attn`（text_i 对 video_j 的 16 个 anchor 的 softmax 注意力）在正负对上分布相近。软最大化（logit_scale≈14.3）使注意力总是集中在 1-2 个 anchor 上，熵始终很低，MLP 无法从输入区分正负对

### 为什么 anchor_attn 不区分正负对

```
text_i @ anchors_i[k]:  softmax → [0.31, 0.02, 0.15, ...] 熵 0.35 → attn 尖锐
text_i @ anchors_j[k]:  softmax → [0.28, 0.05, 0.10, ...] 熵 0.38 → attn 同样尖锐
```

两组注意力都是低熵的——因为 CLIP 表征空间中，任何文本总会找到与其语义最接近的 1-2 个 anchor。无论视频是否匹配，16 路分布看起来都像一个“峰值 + 长尾”。MLP 收到的是两个几乎相同的 16 维向量。

## 结论

per-pair 置信度路径的通路被彻底探索完毕：

| 变体 | 问题 |
|------|------|
| 无 detach | 梯度冲突，ret_gap 崩到 16.7 |
| detach + MLP | 梯度隔离成功，但输入无区分度，MLP 坍缩 |

确定性结论：**基于 anchor 注意力的 per-pair 置信度不能提升 R@1**。下一个 agent 应放弃此方向，回退到 B1-only（50.0 已验证）。
